Blockcahin
Good idea
Lots of applications & ideas for blockchain have failed
Good idea for an application needs to require ALL the core benefits
For blockchain there are 4 core principles
1. Public
2. Decentralized
3. Immutable
4. Trust
If only need some of these, I believe you will have “pretty good” chance
If need all, then high chance of success
Public
On blockchain, all transactions are public
By typing in the wallet addresses can see all the transactions
For ethereum, the solidity contract is public
Decentralized
The data is not stored on one server
On blockchain there are many (can consider infinite) copies of the data all over the world
No one controls this data/value
Immutable
Immutable means non-changeable
That is a core principle of blockchain versus database which is changeable
On blockchain, the data can be considered as chain links
_ each link is locked into the data before and after it
Trust
Trust means do you REALLY know who the people involved are
Blockchain uses unique wallet addresses
Transactions are protected with high security encryption routines
When a transaction is approved by the blockchain there is no doubt about
_ who the payee is
_ the payee has the correct amount in their wallet
_ who the receiver is
When transfer happens,
_ it is valid transaction
_ completed in seconds
